# CampusActivityManagementSystem
### 软件项目实训选题——校园活动管理系统

------
本仓库为后端实体的CRUD功能代码，所有回传均为数据实体的json数据。

#### 完成的实体类

1.用户类

2.活动类

3.活动报名类

4.签到类

5.物资类

6.物资领用记录类

7.评价反馈类

8.统计类

---
#### 实现的功能

1.用户管理
- 支持用户的增删改查（通过 UserService 接口实现）

2.活动管理

- 活动的创建、查询（按 ID、全部）、更新、删除
- 活动创建时的时间有效性校验（开始时间不能大于等于结束时间）
- 时间格式校验（统一使用 "yyyy-MM-dd HH:mm" 格式）

3.活动报名管理
- 活动报名、查询（按 ID、活动 ID、用户 ID）、更新状态、取消报名
- 重复报名校验、活动存在性校验

4.签到管理
- 签到记录的创建、查询（按 ID、报名 ID、活动 ID、用户 ID）、更新（签到 / 签退）、删除
- 签到时间有效性校验（签退时间不能早于签到时间）
- 重复签到校验、报名记录存在性校验

5.物资管理
- 物资的增删改查
- 支持按类型查询、按名称模糊查询

6.物资领用管理
- 领用记录的增删改查
- 支持按活动 ID、用户 ID、物资 ID 查询领用记录

7.评价反馈管理
- 评价反馈的增删改查
- 支持按活动 ID、用户 ID 查询评价

8.统计分析
- 单个活动的统计数据查询
- 所有活动的汇总统计查询
- 支持批量查询活动统计数据
